# Quickstart

This guide walks you through installing the ELIN toolchain, writing a simple program, compiling it to bytecode, and executing it with the C++ virtual machine.

## Prerequisites

You need two things installed on your machine:

- **Python 3** — to run the compiler
- **A C++ compiler** — to build the virtual machine (`g++` recommended)

<CodeGroup>

```bash check Python
python3 --version
```

```bash check g++
g++ --version
```

</CodeGroup>

## Getting the source

Clone the ELIN repository from GitHub. The repository contains both the compiler and the VM source code.

```bash
git clone https://github.com/schallten/elin
cd elin
```

The repository has two main directories:

```text
elin/
├── compiler/       # Python compiler source
│   ├── main.py
│   ├── compiler.py
│   ├── ops.py
│   └── expression_parser.py
└── executor/       # C++ virtual machine
    └── main.cpp
```

## Writing and running your first program

<Steps>
  <Step title="Write a source file">
    Create a file called `hello.elin` in the root of the repository:

    ```elin hello.elin
    let x = 42
    print x
    halt
    ```

    Every ELIN program follows this pattern:
    - `let <name> = <expression>` declares and assigns a variable.
    - `print <name>` prints the value of a variable.
    - `halt` stops execution.

    <Warning>
      Every variable you declare must be used at least once. The compiler will
      reject programs with unused variables.
    </Warning>
  </Step>

  <Step title="Compile to bytecode">
    Run the compiler from the repository root, passing your source file as the argument:

    ```bash
    python3 compiler/main.py hello.elin
    ```

    A successful compilation prints the generated bytecode and writes a `.outz` file alongside it:

    ```text
    ✓ Compilation successful: hello.elin -> hello.outz

    Generated Bytecode:
    # Package: hello
    #
    #
    #
    1 0 0 0 42
    3 0
    8 0
    9
    ```

    The output file `hello.outz` now contains the bytecode. Each line is a single instruction encoded as space-separated integers:

    | Bytecode line | Meaning |
    |---|---|
    | `1 0 0 0 42` | PUSH the integer `42` onto the stack |
    | `3 0` | STORE the top of stack into variable slot `0` (x) |
    | `8 0` | PRINT the value in variable slot `0` |
    | `9` | HALT — stop execution |

    <Note>
      Lines beginning with `#` are header comments that the VM skips. The
      compiler always emits four header lines (`# Package: <name>` followed by
      three blank comment lines).
    </Note>
  </Step>

  <Step title="Build the virtual machine">
    Compile the C++ executor with `g++`:

    ```bash
    g++ -o elin executor/main.cpp
    ```

    This produces an `elin` binary in the current directory.

    <Tip>
      You only need to build the VM once. Recompile it only if you modify
      `executor/main.cpp`.
    </Tip>
  </Step>

  <Step title="Run your program">
    The current VM reads from a file named `test.outz` in the working directory.
    Copy your compiled output to that name, then run the executor:

    ```bash
    cp hello.outz test.outz
    ./elin
    ```

    Expected output:

    ```text
    === Execution Started ===
    42
    === Execution Finished ===
    ```

    <Note>
      The executor currently hardcodes the filename `test.outz`. To run a
      different program, copy its `.outz` file to `test.outz` before executing,
      or modify the `ifstream file("test.outz")` line in `executor/main.cpp` to
      accept a command-line argument.
    </Note>
  </Step>
</Steps>

## A more complete example

The following program uses arithmetic, a `while` loop, and `if`/`else` branching to count from 1 to 5, printing a special message when it reaches 3.

```elin counter.elin
let i = 1
while i <= 5
    if i == 3
        let msg = 999
        print msg
    else
        print i
    end
    let i = i + 1
wend
halt
```

<Note>
  ELIN is integer-only. There are no string values. The example uses `999` as a
  sentinel integer to represent the "special" case at `i == 3`.
</Note>

Compile and run it:

```bash
python3 compiler/main.py counter.elin
cp counter.outz test.outz
./elin
```

Output:

```text
=== Execution Started ===
1
2
999
4
5
=== Execution Finished ===
```

### How the loop compiles

The `while` loop compiles into a condition check followed by a conditional jump (`JZ` — jump if zero) to skip the body, and an unconditional jump (`JMP`) back to the condition at the end of each iteration:

```text
# Package: counter
#
#
#
1 0 0 0 1      <- PUSH 1
3 0            <- STORE into var[0]  (i = 1)
2 0            <- LOAD var[0]        (condition: i)
1 0 0 0 5      <- PUSH 5
13             <- CMP_LTE (i <= 5)
17 <end>       <- JZ: jump past loop if condition is false
...            <- loop body
16 2           <- JMP back to condition check
9              <- HALT
```

<Tip>
  The full `test.elin` included in the repository (inside `compiler/`) is a
  more complex example that loops from 1 to 10,000 and uses nested
  `if`/`else` blocks. It is a good reference for seeing how the compiler
  handles deep nesting.
</Tip>

## Language quick reference

| Statement | Example | Notes |
|---|---|---|
| Variable declaration | `let x = 10` | Must be used at least once |
| Assignment with expression | `let z = x + y * 2` | Operator precedence applies |
| Print | `print x` | Prints variable value, not literals |
| Conditional | `if x > 5 ... end` | Requires `end` to close |
| Conditional with else | `if x > 5 ... else ... end` | `else` at the same indentation level |
| While loop | `while i <= 10 ... wend` | Use `wend` to close |
| Stop program | `halt` | Compiler appends one automatically if missing |
| Comment | `# comment` or `// comment` | Both styles are supported |
