# Bytecode format

The ELIN compiler produces `.outz` files — plain-text files of space-separated integers, one instruction per line, preceded by a short header. The virtual machine reads these files at startup, skips the header, and loads the remaining lines into an indexed instruction array.

## File structure

A `.outz` file consists of two sections:

1. **Header** — Four lines beginning with `#`. These are ignored by the VM.
2. **Instructions** — One instruction per line, each a sequence of space-separated integers.

```text example.outz
# Package: example
#
#
#
1 0 0 0 42
3 0
8 0
9
```

### Header format

```text
# Package: <package_name>
#
#
#
```

The package name is taken from the stem of the source filename (e.g. `hello.elin` → `hello`). The three blank comment lines that follow are structural placeholders reserved for future metadata.

## Instruction encoding

Every instruction starts with an opcode integer. Some instructions carry operands as additional integers on the same line.

### Instruction formats

| Opcode | Mnemonic | Format | Notes |
|---|---|---|---|
| `1` | `PUSH` | `1 0 0 0 <value>` | Literal integer push. Value is always at token index 4. |
| `2` | `LOAD` | `2 <var_index>` | Load variable slot onto the stack. |
| `3` | `STORE` | `3 <var_index>` | Pop stack top and store into variable slot. |
| `4` | `ADD` | `4` | Pop two, push sum. |
| `5` | `SUB` | `5` | Pop two, push difference (`a - b`). |
| `6` | `MUL` | `6` | Pop two, push product. |
| `7` | `DIV` | `7` | Pop two, push quotient (`a / b`). |
| `8` | `PRINT` | `8 <var_index>` | Print value of variable slot to stdout. |
| `9` | `HALT` | `9` | Terminate execution. |
| `10` | `CMP_EQ` | `10` | Pop two, push `1` if equal, else `0`. |
| `11` | `CMP_NEQ` | `11` | Pop two, push `1` if not equal, else `0`. |
| `12` | `CMP_LT` | `12` | Pop two, push `1` if `a < b`, else `0`. |
| `13` | `CMP_LTE` | `13` | Pop two, push `1` if `a <= b`, else `0`. |
| `14` | `CMP_GT` | `14` | Pop two, push `1` if `a > b`, else `0`. |
| `15` | `CMP_GTE` | `15` | Pop two, push `1` if `a >= b`, else `0`. |
| `16` | `JMP` | `16 <address>` | Unconditional jump to address. |
| `17` | `JZ` | `17 <address>` | Jump to address if stack top is `0` (pop). |
| `18` | `JNZ` | `18 <address>` | Jump to address if stack top is not `0` (pop). |

### Special PUSH encoding

`PUSH` uses a five-token format to accommodate future extensions. The three zeros at positions 1–3 are currently unused padding:

```text
1 0 0 0 <value>
```

The VM reads the value from `tokens[4]`.

## Jump addresses

<Note>
Jump addresses are **0-indexed positions into `bytecode_program`** — the array of non-comment, non-blank instructions loaded by the VM. They do not correspond to line numbers in the `.outz` file; header lines and blank lines are skipped before indexing begins.
</Note>

For example, if the first four lines of a `.outz` file are the header, the first instruction line gets address `0`, the second gets `1`, and so on. A `JMP 4` instruction transfers control to the fifth instruction in the program array.

## Worked example

This ELIN program prints integers from `1` to `3`:

```elin counter.elin
let i = 1
let limit = 3
while i <= limit
  print i
  let i = i + 1
wend
halt
```

The compiler produces the following `.outz` file:

```text counter.outz
# Package: counter
#
#
#
1 0 0 0 1
3 0
1 0 0 0 3
3 1
2 0
2 1
13
17 14
8 0
2 0
1 0 0 0 1
4
3 0
16 4
9
```

### Annotated walkthrough

| Address | Instruction | Meaning |
|---|---|---|
| `0` | `1 0 0 0 1` | PUSH literal `1` |
| `1` | `3 0` | STORE into `i` (slot 0) |
| `2` | `1 0 0 0 3` | PUSH literal `3` |
| `3` | `3 1` | STORE into `limit` (slot 1) |
| `4` | `2 0` | LOAD `i` (slot 0) — loop condition start |
| `5` | `2 1` | LOAD `limit` (slot 1) |
| `6` | `13` | CMP_LTE — push `1` if `i <= limit`, else `0` |
| `7` | `17 14` | JZ 14 — exit loop if condition is `0` (jumps to HALT) |
| `8` | `8 0` | PRINT `i` (slot 0) |
| `9` | `2 0` | LOAD `i` (slot 0) — begin `i = i + 1` |
| `10` | `1 0 0 0 1` | PUSH literal `1` |
| `11` | `4` | ADD |
| `12` | `3 0` | STORE into `i` (slot 0) |
| `13` | `16 4` | JMP 4 — back to loop condition |
| `14` | `9` | HALT |

<Note>
The `JZ` at address `7` targets address `14` — the `HALT` instruction. When the loop condition becomes false (`i > limit`), the VM jumps directly to `HALT` and terminates. The `JMP` at address `13` jumps back to address `4` to re-evaluate the condition at the start of the next iteration.
</Note>

### Variable slot assignment

The compiler assigns slots in the order variables are first defined:

| Slot | Variable |
|---|---|
| `0` | `i` |
| `1` | `limit` |
