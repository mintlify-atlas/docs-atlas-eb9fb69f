# Compiler

The ELIN compiler is a single-pass Python compiler that reads a `.elin` source file and produces a `.outz` bytecode file. It lives in the `compiler/` directory and is invoked directly with Python.

## Usage

```bash
python3 compiler/main.py <source_file.elin>
```

The compiler derives the output filename from the source file's stem. A file named `hello.elin` produces `hello.outz` in the current working directory. After a successful compilation, the compiler prints a confirmation message and the full generated bytecode to stdout.

```bash
$ python3 compiler/main.py hello.elin
✓ Compilation successful: hello.elin -> hello.outz

Generated Bytecode:
# Package: hello
#
#
#
1 0 0 0 10
3 0
8 0
9
```

<Warning>
If the source file is not found, or a compilation error occurs (undefined variable, unused variable, missing `end`, or unknown command), the compiler prints an error message and exits with code 1.
</Warning>

## Compilation pipeline

<Steps>
  <Step title="Read source file">
    `main.py` reads all lines from the `.elin` source file and passes them to the `Compiler` class along with the package name (derived from the file stem).
  </Step>
  <Step title="Parse and emit bytecode">
    `compile_raw()` iterates over the source lines. When it encounters a `while` or `if` keyword it calls `_collect_block()` to gather the entire block before dispatching to the appropriate handler. All other statements are dispatched to `compile_line()`.
  </Step>
  <Step title="Append HALT">
    After `compile_raw()` returns, `compile()` checks whether the last emitted instruction is already a `HALT` (`9`). If not, it appends one automatically.
  </Step>
  <Step title="Check for unused variables">
    `check_unused_variables()` scans the `variables` dict and aborts compilation if any variable was defined but never read.
  </Step>
  <Step title="Write output">
    A four-line header is prepended to the bytecode list and the result is joined with newlines. `main.py` writes this string to `<stem>.outz`.
  </Step>
</Steps>

## The `Compiler` class

The `Compiler` class (`compiler/compiler.py`) holds all compilation state and exposes one public entry point: `compile(lines)`.

### Variable tracking

The compiler tracks every variable in a dictionary:

```python
self.variables = {}  # name -> {'index': int, 'defined': bool, 'used': bool}
```

Each variable is assigned a sequential integer slot starting at `0` (`next_var_index`). This slot is used directly in `LOAD`, `STORE`, and `PRINT` instructions. The `defined` flag is set when a `let` statement first declares the variable; the `used` flag is set when the variable appears in an expression or a `print` statement.

### Key methods

#### `compile_raw(lines)`

The main parsing loop. It walks the line list with an explicit index `i` rather than a `for` loop so that it can skip entire blocks at once. When it sees `while` or `if`, it calls `_collect_block()`, advances `i` past the block, and then dispatches to `handle_while()` or `handle_if()`. Every other non-blank, non-comment line goes to `compile_line()`.

#### `_collect_block(lines, start_idx, open_tag, close_tag)`

Collects all lines belonging to a structured block by tracking nesting depth. It starts at `depth = 1` and increments when it sees `open_tag` (e.g. `while`) and decrements when it sees `close_tag` (e.g. `wend`). Returns the collected lines and the index immediately after the closing tag.

#### `handle_assignment(segments)`

Handles `let` statements. The target variable is registered (or looked up) with `define_variable()`, which assigns it the next available slot. The right-hand side expression (everything after `=`) is then scanned for a comparison operator. If one is found the expression is split into left-hand side, operator, and right-hand side, each compiled separately before the comparison opcode is emitted. Otherwise `_compile_expression()` handles the arithmetic. A `STORE` instruction targeting the variable's slot is emitted last.

#### `handle_print(segments)`

Handles `print` statements. If the argument is a literal integer, a hidden temporary variable (`__literal_<value>`) is created, the literal is pushed and stored, and the `PRINT` instruction references that slot. If the argument is a variable name, `use_variable()` looks up its slot directly. Emits `8 <var_index>`.

#### `handle_while(while_condition, while_body)`

Emits a loop by recording the current bytecode position as `start_addr`, then compiling the condition via `handle_comparison()`. A `PLACEHOLDER_JZ` is appended and its position recorded as `jz_pos`. The body is compiled recursively with `compile_raw()`. A `JMP` back to `start_addr` is appended. Finally, `bytecode[jz_pos]` is patched with `17 <end_addr>` — the instruction after the `JMP` — so the loop exits when the condition is zero (false).

#### `handle_if(lines, has_else)`

Compiles `if`/`else`/`end` blocks. It scans the block lines to locate `else` and `end` positions while tracking nesting depth for inner `if` blocks.

- **Without `else`**: Emits `PLACEHOLDER_JZ`, compiles the body, then patches the placeholder with `17 <end_label>`.
- **With `else`**: Emits `PLACEHOLDER_JZ`, compiles the if-body, emits `PLACEHOLDER_JMP`, records `else_label`, compiles the else-body, then patches both placeholders: `JZ` jumps to `else_label` and `JMP` jumps to `end_label`.

#### `check_unused_variables()`

Scans `self.variables` after full compilation. Any entry where `used` is `False` is collected into an error list. If the list is non-empty, the compiler prints all unused names and exits with code 1.

### Expression parsing — `infix_to_postfix()`

Multi-token arithmetic expressions are converted from infix to postfix (Reverse Polish Notation) by `infix_to_postfix()` in `compiler/expression_parser.py`. The function implements the **Shunting Yard algorithm**:

- Operands (numbers and variable names) go directly to the output queue.
- Operators (`+`, `-`, `*`, `/`) are pushed onto an operator stack, but only after popping any operators of greater or equal precedence to the output first.
- `(` is pushed onto the stack; `)` flushes operators back to output until the matching `(`.

Precedence is defined as `*` and `/` at level 2, `+` and `-` at level 1. The resulting postfix token list is then walked by `_compile_expression()`, which calls `parse_operand()` for values and `add_op()` for operators.

## Error handling

| Condition | Error message |
|---|---|
| Undefined variable used in expression | `Error: Variable '<name>' used before definition.` |
| Variable defined but never used | `Error: Unused variables: <name>, ...` |
| `if` block with no matching `end` | `Error: Missing 'end' for 'if'` |
| Unknown statement keyword | `Error: Unknown command '<cmd>'` |
| Source file not found | `Error: File '<path>' not found.` |

All errors cause an immediate `sys.exit(1)`.

## Example

Given this source file:

```elin counter.elin
let i = 1
let limit = 3
while i <= limit
  print i
  let i = i + 1
wend
halt
```

The compiler produces:

```text counter.outz
# Package: counter
#
#
#
1 0 0 0 1
3 0
1 0 0 0 3
3 1
2 0
2 1
13
17 17
8 0
2 0
1 0 0 0 1
4
3 0
16 4
9
```

Breaking it down:

- Lines 5–6: Push `1`, store in `i` (slot 0).
- Lines 7–8: Push `3`, store in `limit` (slot 1).
- Lines 9–11: Load `i` and `limit`, emit `CMP_LTE` (opcode `13`) — the loop condition.
- Line 12: `JZ 17` — exit loop if condition is zero.
- Line 13: `PRINT 0` — print `i`.
- Lines 14–16: Load `i`, push `1`, `ADD`, store back into `i`.
- Line 17: `JMP 4` — jump back to the loop condition.
- Line 18: `HALT`.
